Async Testing

Testing Promises and Async Code

Fauji supports async tests and matchers for promises. You can use async/await or return a promise from your test function.

test('resolves to 5', async () => {
  await expect(Promise.resolve(5)).resolves.toBe(5);
});

test('rejects with error', async () => {
  await expect(Promise.reject(new Error('fail'))).rejects.toThrow('fail');
});

Async Matchers

  • resolves: Asserts a promise resolves to a value
  • rejects: Asserts a promise rejects with a value/error
  • toResolve: Asserts a promise resolves (no value check)
  • toReject: Asserts a promise rejects (no value check)
// resolves
await expect(Promise.resolve(42)).resolves.toBe(42);

// rejects
await expect(Promise.reject('fail')).rejects.toBe('fail');

// toResolve
await expect(Promise.resolve()).toResolve();

// toReject
await expect(Promise.reject()).toReject();

Async Test Functions

You can use async functions or return a promise in your tests:

test('async/await example', async () => {
  const data = await fetchData();
  expect(data).toBeDefined();
});

test('promise example', () => {
  return somePromise().then(result => {
    expect(result).toBe(true);
  });
});

Tips

  • Always await or return the promise in your test to ensure Fauji waits for completion.
  • Use resolves and rejects for expressive async assertions.
