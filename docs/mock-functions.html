    Mock Functions

    Overview

    Fauji provides built-in utilities for spying, stubbing, and mocking functions and modules. These tools help you track calls, replace implementations, and isolate dependencies in your tests.

    Spies

    A spy wraps a function and tracks its calls and arguments.

    const { spy } = require('fauji/matchers');
  const fn = spy();
  fn(1, 2);
  expect(fn.callCount).toBe(1);
  expect(fn.calls[0]).toEqual([1, 2]);
    Spies can be used to verify how many times a function was called and with what arguments.

    Stubs

    A stub replaces an object's method with a spy, optionally with a custom implementation.

    const obj = { greet: (name) => 'Hello ' + name };
  const { stub } = require('fauji/matchers');
  stub(obj, 'greet', (name) => 'Hi ' + name);
  expect(obj.greet('Sam')).toBe('Hi Sam');
    Stubs can be restored to the original method using the restore method on the returned spy.

    Mocks

    A mock replaces a module's exports with a mock implementation.

    const { mock } = require('fauji/matchers');
  const m = mock('./some-module', { foo: () => 42 });
  // ... use mocked module ...
  m.restore(); // Restore original
    Mocks are useful for isolating dependencies and controlling module behavior in tests.

    Tips

    • Use spies to verify function calls and arguments.
    • Use stubs to replace methods for specific test scenarios.
    • Use mocks to isolate modules and control their exports.
